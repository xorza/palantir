# Aperture

A Rust GUI crate. **Immediate-mode authoring API**, **WPF-contract two-pass layout with flex-shrink sizing**, **wgpu rendering**.

Read the **Architecture** section below for the full design rationale before making non-trivial changes.

## Posture

State-of-the-art UI framework, craft-driven. **No external consumers, no published API** тАФ treat it as sports programming.

- **Break things freely.** Rename, refactor, big-bang migrations welcome тАФ no deprecation shims, compat aliases, feature flags, or migration helpers. Bar is "fmt + clippy + tests pass and the showcase still feels right by eye."
- **Per-frame allocation is a real metric.** Steady-state must be heap-alloc-free after warmup. New per-frame `Vec::new()` / `HashMap` rebuild = regression; push onto retained scratch with capacity reuse.
- **API ergonomics matter.** Builder chains read like prose, defaults are right, surprise behavior gets a pinning test. When in doubt, prioritize call-site readability.
- **Optimize aggressively when motivated.** Micro-wins (struct packing, const fns, scratch reuse, cache layout) are encouraged even without a workload demanding them.
- **Ship in measurable slices.** One feature with tests + a showcase section (on the matching tab тАФ tabs group related features, e.g. all form controls share `controls`) beats a half-finished cluster. If a change is structurally complex with no motivating workload, say "too early" and shelve with a note rather than ship speculation.
- **Docs are starting positions, not commitments.** Treat `docs/*.md`, colocated module docs, and this file as evolving and possibly wrong. When a doc contradicts user intent or current code, double-question rather than defer тАФ flag the conflict, ask, and update the doc.

## Code style

- **Inherent impls live with their struct.** An `impl Foo { тАж }` block belongs in the same file as `struct Foo`. Only **trait** impls (`impl Trait for Foo`) may live elsewhere (e.g. beside the trait, or in a submodule). When splitting a type's behavior across files, move the *helper types + free functions* it calls into the sibling (exposing them `pub(crate)`), not the inherent methods тАФ so everything you can call on `Foo` is found next to `Foo`. Example: `ImagePipeline::paint_gpu_views` stays in `image_pipeline/mod.rs` with the struct; the `RenderTarget` value type + the `make_target` allocator it drives live in `image_pipeline/render_target.rs`.
- **"Non-paintable scalar" predicate:** when guarding on a magnitude (stroke width, alpha, radius тАФ any scalar where тЙд 0, NaN, or near-zero means "skip / emit nothing"), use `crate::primitives::approx::noop_f32(v)`. It captures all three cases in one comparison and is the shared predicate behind `Stroke::is_noop` / `Color::is_noop` / `Shape::is_noop`. Don't hand-roll `if !(v > 0.0)` or `if v <= 0.0 || v.is_nan()` тАФ they drift apart over time. `approx::EPS = 1e-4`; for sub-`EPS` thresholds use the constant directly.
- **Tests in `lib.rs` pin layout semantics.** Add a test whenever you change measure/arrange behavior. Don't add wgpu code paths to the layout/tree modules.
- **`bytemuck::Pod` structs use `#[padding_struct::padding_struct]`.** The proc macro injects trailing padding fields so the struct's size is a multiple of its alignment, satisfying `Pod`'s no-padding-bytes invariant. Don't hand-add `_pad: u32` fields тАФ they rot when a new field shifts the layout. Construct via `Self { real_field: x, ..bytemuck::Zeroable::zeroed() }` so the spread fills whatever padding the macro generated; `unsafe { std::mem::zeroed() }` for `const` sentinels. Existing examples: `DrawPolylinePayload` / `DrawMeshPayload` (`src/renderer/frontend/cmd_buffer/mod.rs`), `TextCacheKey` (`src/text/mod.rs`).
- **`#[track_caller]` is reserved for automatic widget identity.** Use it only on `WidgetId::auto_stable` and widget constructors, builders, or forwarding helpers that must carry the authoring call site to that function. Do not use it for panic attribution, tracing, test assertions, or non-widget value constructors.
- **`WidgetId`** is hashed from a user-supplied key тАФ keep IDs stable across frames so persistent state survives. Auto-deriving constructors (`Button::new`, `Text::new`, `Panel::hstack`, тАж) use `WidgetId::auto_stable()` + `#[track_caller]` so calls at different source lines get distinct ids. `#[track_caller]` does **not** propagate through closure bodies, so a helper that builds widgets inside a closure passed to e.g. `Panel::show(ui, |ui| { ... })` resolves every call to the same source location тАФ but `Ui::widget_id` resolves auto ids **parent-scoped** (mixed with the opened parent's resolved id, exactly like `.id_salt(k)`) and then disambiguates any remaining *same-parent* collision with a per-id occurrence counter, so loops and closure helpers Just Work. Because identity tracks tree position rather than global record order, a subtree drawn from a helper under a stable-id parent keeps stable ids even when whole siblings reorder in paint order (e.g. raising a graph node), so a reorder can't re-key state or spuriously damage untouched subtrees. The occurrence counter stays positional only *within one parent's sibling list*, so reordering or conditionally inserting a sibling at the same call site under the same parent re-keys the affected occurrences тАФ override with `.id_salt(key)` (the builder method on `Configure`) where `key` is something stable like the item's domain id. Explicit-key collisions are caller bugs: `SeenIds::resolve` disambiguates them the same way as auto ids and pushes a `CollisionRecord` onto `forest.collisions`. After the regular paint walk, the encoder (`encoder::emit_collision_overlays`) emits a magenta 3px stroke quad over each colliding node's arranged rect тАФ unclipped, on top of every layer. Always on, no opt-in flag.

## Architecture

Five passes per frame on an arena `Tree` rebuilt every frame (with `tree.post_record` finalizing `subtree_end` + per-node + subtree-rollup hashes between record and measure):

1. **Record** тАФ user code (`Button::new().label("x").show(&mut ui)`) appends per-node columns + `Shape`s.
2. **Measure** (post-order) тАФ node returns desired size given available; `MeasureCache` short-circuits whole subtrees on `(WidgetId, subtree_hash, available_q)` hits. Single dispatch (no WPF-style grow loop).
3. **Arrange** (pre-order) тАФ parent assigns final `Rect` to each child.
4. **Cascade** (pre-order) тАФ `CascadesEngine::run` flattens disabled/invisible/clip/transform and builds the hit index in the same walk, producing a frozen `Cascades` result (`src/ui/cascade/`) consumed by damage diff, hit-test, _and_ the encoder so they can't drift.
5. **Encode + Compose + Paint** тАФ `Encoder` walks the tree and emits a `RenderCmdBuffer` from scratch each frame; `Composer` groups by scissor, snaps to physical pixels; `WgpuBackend` submits instanced quads. `Damage` returns `Full` / `Partial(rect)` / `Skip` and filters which leaves the encoder paints. No encode or compose caches тАФ both were implemented and removed after profiling; the encoder + composer are already memcpy-shaped and a per-frame rebuild beat a per-subtree cache replay.

Widget _state_ (scroll offset, text cursor, animation) lives in `StateMap` on `Ui`: per-type dense `Vec<T>` stores keyed by `WidgetId` (one box per distinct `T`, swap-remove sweep тАФ no per-row `Box`, no `Any` downcast on the hot path). Access via `Ui::state_mut::<T>(id)`; rows for `WidgetId`s not recorded this frame are dropped in `finalize_frame` via the same `removed` slice that `Damage`, `TextReuseCache`, and `MeasureCache` consume.

**Text buffer cache.** `TextShaper` owns one app-global content cache of cosmic-text `Buffer`s keyed by canonicalized `TextCacheKey` shaping inputs. Each window's `LayoutEngine` separately owns its `TextReuseCache`, keyed by `(WidgetId, ordinal)`, so identical ids in different windows cannot overwrite or evict each other's identity rows. The global buffer cache is an ordinary bounded LRU: layouts and per-window reuse rows may retain keys after their buffers are evicted. Before emitting each visible text command, the frontend resolves the retained `ShapeRecord::Text` through its window's `RecordStore` and calls `ensure_buffer`; a miss reconstructs the exact buffer from the key's canonical 1/64 px values. Clip/ellipsis reconstruction restores its unbounded probe internally. The backend's missing-key panic remains the invariant check that every submitted text run passed through the frontend boundary. There is no window registration, owner pin set, or per-key liveness state.

**Tree = SoA columns indexed by `NodeId.0`:** `records: Soa<NodeRecord>` (via `soa-rs`) packs five per-node columns тАФ `widget_id` (hit-test + state map + damage), `shape_span: Span` (slice into the flat shape buffer covering this node's subtree), `subtree_end: SubtreeEnd` (a `u32` newtype; pre-order skip тАФ `i + 1 == subtree_end` for a leaf тАФ every walk), `layout: LayoutCore` (mode/size/padding/margin/align/visibility, bundled because measure reads all six together), `attrs: NodeFlags` (2-byte packed sense/disabled/clip/focusable тАФ cascade/encoder). Adjacent on the tree but outside the SoA: `shapes: Shapes` (flat per-frame `ShapeRecord` buffer; variable-length payloads for `Polyline`/`Mesh`/gradients live on `RecordStore`) and a packed per-node `extras_idx: Vec<ExtrasIdx>` whose three `Option<Index16>` fields (`bounds`, `panel`, `chrome`) use the nonzero niche to remain two bytes while indexing dense `bounds_table: Vec<BoundsExtras>` (transform / position), `panel_table: Vec<PanelExtras>` (grid cell, scroll axes), and `chrome_table: Vec<ChromeRow>` (panel chrome **plus** mask radius for `ClipMode::Rounded` тАФ a `ChromeRow` is allocated even when paint is `is_noop` so the encoder can read the radius for the stencil-mask path). `paint_anims: PaintAnims` is a shape-keyed registry for paint-only animations (alpha mods today); `rollups: SubtreeRollups` carries per-node + subtree hashes and the sparse non-leaf direct-text owner set, populated together in `post_record`. soa-rs lays each `NodeRecord` field out as its own contiguous slice, so each pass touches only the columns it needs. Atomic push across the SoA columns means `open_node` writes all five per-node fields together тАФ they can't drift. Measured `desired`/`rect`/`text_shapes`/`scroll_content`/`available_q` live on `LayoutResult` keyed by `NodeId`, not on the tree.

**Cross-frame work-skip cache.** `MeasureCache` (`src/layout/cache/`) is keyed on `(WidgetId, subtree_hash, available_q)`. A hit blits last frame's subtree (`desired` + `text_shapes`) and skips recursion. The `removed` sweep evicts it alongside `StateMap`, `AnimMap`, and `TextReuseCache`. Encode and compose ran the same keying historically but contributed <1% of frame time; both were removed. **`Damage`** is `enum Damage { Skip, Full, Partial(DamageRegion) }`; `Damage::Skip` is the "nothing changed, just present" skip signal. `Ui::invalidate_prev` rewinds the prev-frame snapshot when the host failed to actually present.

**Layered recording.** `Forest` (`src/forest/mod.rs`) holds one `Tree` per `Layer` variant (`Main`/`Popup`/`Modal`/`Tooltip`/`Debug`); `Ui::layer(layer, anchor, body)` switches the active arena for the body's duration. Pipeline passes iterate `Layer::PAINT_ORDER` bottom-up for paint and reverse for hit-test (topmost-first, so popups reject pointers without per-node z-index). `Popup` widget (`src/widgets/popup.rs`) is the canonical consumer.

**`Shape`** (paint primitive: `RoundedRect`, `Triangle`, `Line`, `Polyline`, `CubicBezier`, `QuadraticBezier`, `Arc`, `Text`, `Mesh`, `Image`, `Shadow`) lowered at authoring time into `ShapeRecord`s in `Tree.shapes.records`, sliced per-node via `records.shape_span()[i]` (a `Span` into the buffer); variable-length payloads (polyline points/colors, mesh verts/indices, gradients) live on `RecordStore`. `RoundedRect` always paints the owner's full arranged rect тАФ no per-shape positioning. Text is the sole layout-aware variant: leaf-owned text shapes during measure and contributes desired size; direct container text is paint-only and shapes after arrange against the final padded width. Direct container-text owners are discovered during the existing finalized-tree traversal, so layout shapes only those owners after arrange. All geometric shapes remain invisible to layout. Paint consumes the layout-derived text key/extent but otherwise ignores hierarchy beyond `subtree_end`. **This boundary is load-bearing тАФ geometric paint must not start driving layout.**

**`GpuView`** (raw `wgpu` into a widget). App code implements `GpuPaint` (`src/renderer/gpu_view.rs`) on its own renderer, wraps it `Rc<RefCell<тАж>>`, and hands it to `GpuView::new(paint)`. The framework owns an `Rgba8UnormSrgb` off-screen texture sized exactly to the widget's physical rect, runs the callback into it during submit, and **composites it as an ordinary image** тАФ same shared `TextureIdSource`, same image draw тАФ so clipping, rounded corners, z-order, and partial-damage recompositing come for free. **This "GpuView is an image" is load-bearing тАФ decoupling the draw would re-implement the compositor.** Data flow, no `Ui`-side registry beyond one map: `Ui::gpu_view` upserts `Ui::gpu_views: FxHashMap<WidgetId, GpuViewEntry>` (the entry holds the stable `texture_id` minted once from `RenderAssets`, the `paint` callback refreshed each frame, and an `epoch`), then records `ShapeRecord::GpuView { epoch }` тАФ *only* the epoch rides the shape. The encoder atomically emits a typed image command with its optional paint callback; the composer lists each view in `RenderBuffer.frame_targets` (`id` + used px + callback), since physical size only exists post-compose. Backend `ImagePipeline::paint_gpu_views` delegates allocation, resize, callback execution, and owner-scoped eviction to `image_pipeline/render_target.rs`; texture uploads and bind groups live separately in `image_pipeline/textures.rs`. **`epoch` is the damage version:** bumped to the frame id on `repaint(true)` (default), held stable on `.repaint(false)` so the shape hash doesn't change тЖТ the view is undamaged тЖТ the encoder culls it and its GPU paint is skipped (a static view idles when neighbors force frames). **Eviction is immediate and owner-scoped** тАФ a target owned by the submitting render stream but absent from its `frame_targets` is freed; other windows' targets survive. The catch: a culled `repaint(false)` view's texture is freed, so `GpuPaint::init` re-runs when it next composites (guard expensive setup). The texture format / id minting / immediate-eviction tradeoff are documented on the types; `tests/visual/fixtures/gpu_view.rs` + `src/bin/showcase/cube.rs` are the worked examples.

**Colour pipeline.** Linear-RGB f32 everywhere on the CPU side; sRGB encoding happens on the GPU at swapchain write. Specifically: `Color { r, g, b, a: f32 }` (`src/primitives/color.rs`) stores **straight-alpha linear-RGB** values in 0..1 (or >1 for HDR-shaped tween outputs). User-facing constructors `Color::rgb(r,g,b)` / `Color::hex(0x...)` / `Color::rgb_u8(...)` interpret their input as **sRGB perceptual** and linearise via a cubic Hejl-Burgess-Dawson fit (`srgb_to_linear`, max error ~1.5e-3 тАФ pinned by `cubic_srgb_max_error_under_two_thousandths`). `Color::linear_rgb` / `Color::linear_rgba` skip the linearisation for tween outputs and physically-derived values. Storage in `Background`, `Stroke`, `Brush::Solid`, `Quad`, etc. is always linear. All AA / blend / `Animatable::lerp` math runs in linear. The render surface is configured to an sRGB texture format (`is_srgb()` pick in `src/host/winit/gpu.rs`); **every pipeline** (quad / mesh / image) uses `BlendState::PREMULTIPLIED_ALPHA_BLENDING`. **Shader contract: straight-alpha linear in, premultiplied linear out** тАФ `From<Color> for ColorU8` is a straight-alpha quantize, so vertex / instance colours arriving at the shader are straight (`rgb`, `a` independent); the fragment shader writes `vec4(rgb * a, a)` so the premul blend equation receives correctly-shaped source. The GPU does the linearтЖТsRGB encode automatically because the render target is sRGB-format. **Don't write sRGB-encoded values into `Color`** (skips the linearisation contract); use `Color::rgb`/`hex` for sRGB-perceptual input, `linear_rgb` for already-linear input. `ColorU8` (`src/primitives/color.rs`) is a 4-byte **linear-u8** storage type for places where 8-bit linear precision suffices and footprint matters (currently `Stop.color` in gradients). Default `From<Color>` / `From<ColorU8>` are straight linear quantize pairs тАФ no sRGB encode. For the sRGB-encoded form (CSS-style hex input) call `Color::to_srgb_u8` or use `ColorU8::hex` / `hexa` explicitly. The gradient LUT atlas uses `Rgba16Float` (linear, no auto-decode) with `ColorF16` row texels (`gradient_atlas::LutRowTexels`); the shader samples and handles premul directly. f16, not 8-bit linear: a dark stop linearises to a tiny value (`#1a1a2e`'s red тЙИ 3/255), so an 8-bit *linear* row crushes the dark half of a darkтЖТbright gradient onto ~16 integer levels and bands visibly тАФ f16's ~11-bit mantissa at that magnitude keeps the row smooth (pinned by `gradient_atlas::tests::dark_gradient_row_has_no_banding`).

**Sizing (flex-shrink with min-content floor):** `Fixed(n)` = exactly `n` (hard contract; can exceed parent's available). `Hug` = `min(content, available)` floored at `intrinsic_min`. `Fill(weight)` = `available` floored at `intrinsic_min`; with Fill siblings, each gets `leftover * weight / total_weight`, but a sibling whose floor exceeds its share _freezes_ at floor and the rest re-divide (CSS Flexbox-style). The `intrinsic_min` floor is the largest non-shrinkable thing on this axis (Fixed descendant, explicit `min_size`, longest unbreakable word). Children clamp DOWN to fit parent тАФ no WPF-style parent-grow. Overflow only happens when rigid descendants don't fit; downstream tolerates it. **A stack measures its children against its own committed main extent, not `тИЮ`** тАФ so a `Fixed`/`max_size` bound on any ancestor flows down and constrains descendants (CSS `max-height`/`max-width` semantics, not WPF's free-stacking-axis); an _unbounded_ stack still passes `тИЮ` on its main axis, so children report their natural main size and the stack grows to fit. This is what lets a nested `WrapVStack`/`WrapHStack` wrap (or a `Scroll` bound) against a capped ancestor without a cap of its own. Canonical impl: `resolve_axis_size` in `src/layout/support.rs` + the Pass-1 `main_avail` measure + freeze loop in `src/layout/stack/mod.rs::measure`. Pinned by `src/layout/{stack,wrapstack,zstack,canvas,grid}/tests.rs` and `src/layout/cross_driver_tests/convergence.rs`.

**Stack vs. Grid Fill тАФ same contract, deliberately different solvers.** Both resolve `Fill` as "weighted leftover, each child clamped to `[intrinsic_min, max]`, violators freeze and the rest re-divide" тАФ but with different freeze cadences: `stack::freeze_distribute` freezes *every* violator per pass, `grid::resolve_axis` Phase 3 uses constraint-by-exclusion. For *mixed* min/max violations on the same axis the two can converge to different sizes. This is an **accepted, hand-synced divergence** (see the doc comment on `stack::freeze_distribute`), not a bug to silently "DRY up": a shared solver would change one driver's edge-case output, so it needs a deliberate target-semantics decision first, not a refactor. The common cases (all-min or all-max violations, single Fill, no violations) agree.

- `src/text/` тАФ `TextShaper` (app-global cosmic-text measurement + reconstructible bounded shaped-buffer LRU) and `TextReuseCache` (per-window `(WidgetId, ordinal)` identity reuse, owned by `LayoutEngine`) + the rendering glue against `src/renderer/backend/text/`; mono fallback for headless

Per-frame hit lookup goes through the frozen `Cascades` result directly тАФ `Ui::on_input(event, &Cascades)` calls `cascades.hit_test*`; no separate `HitIndex` type
- `src/renderer/` тАФ `frontend/` (encoder + typed SoA command stream in `cmd_buffer/` + composer) and `backend/` (wgpu scheduling/pipelines, shared shader specialization in `shader_template.rs`, custom text backend, and split image textures / `GpuView` targets in `image_pipeline/`). Shared data lives in `render_buffer/` (`RenderBuffer`, batch scheduling records, owner identity), `gradient_atlas/` (CPU atlas, bake logic, shared handle), `damage.rs` (shared padding/culling policy), `image_registry.rs`, `gpu_view.rs`, `texture_id.rs`, and `assets.rs`. ALL strokes (lines / beziers / arcs / polylines) render natively on the GPU curve pipeline тАФ polylines as segment + join-chrome instances (`curve.wgsl`); there is no CPU stroke tessellation

Per-widget tests are colocated as `widgets/<name>/{mod.rs, tests.rs}` pairs or inline `#[cfg(test)]` mods тАФ there is no `widgets/tests/` aggregate; cross-cutting contracts live with the subsystem they pin (drag response composition in `input/tests/drag.rs`, `auto_id` call-site stability in `forest/element/tests.rs`)

- `src/common/` тАФ shared scaffolding: `LiveArena` (cross-frame cache backing, `live_arena.rs`), `Index16` (two-byte capacity-checked arena indices with an `Option` niche), hashing helpers, platform/time shims, `clipboard.rs` (process-wide clipboard: arboard + in-memory fallback, `get`/`set`, used by `TextEdit`). (`RecordStore` lives at `src/record_store.rs`; `PerLayer` lives in `src/forest/`.)
- **Public boundary / test reach-ins:** implementation modules are crate-private; supported consumer types are re-exported only from `lib.rs`. In-tree tests use crate-private colocated `test_support` modules. Every benchmark implementation and shared fixture lives under the production-mirroring `src/bench/` hierarchy; external targets contain only Criterion wiring or a `dhat` global allocator plus one facade call. Integration tests receive only narrow root helpers where an external-crate boundary is itself under test.
- `src/window.rs` тАФ backend-agnostic **multi-window vocabulary** shared by the recorder (`Ui`) and the hosts, deliberately carrying no winit/wgpu types so opening a window from app code doesn't pull a backend into the `Ui` API: `WindowToken` (a caller-chosen opaque `u64` identity тАФ supplied at `Ui::open_window` or host construction, handed to `App::update` and `App::record` through `Ui::frame`, used to address `Ui::close_window` / `HostHandle::request_repaint`) and `WindowConfig` (title + optional logical inner/min size).
- `src/display.rs` тАФ `Display`: per-output state the host facade rebuilds each frame for its internal window driver тАФ surface physical pixel size, DPR scale, pixel-snap flag, monitor refresh rate. Surface-size changes are detected via `logical_rect`, so the other knobs ride along without forcing a relayout. Grouped so future rasterization knobs (sRGB/MSAA/gamma) have a home.
- `src/host/shared.rs` тАФ `HostShared`: the retained app-global authority for `RenderShared` (text shaping + CPU-side `RenderAssets`), `DiagnosticsShared` (debug overlay + latest backend GPU statistics), and `WindowDirectory` (live tokens). It derives narrow `UiShared` and `BackendShared` capability bundles; consumers clone only the `Rc`-backed handles they need.
- `src/host/window_driver.rs` тАФ `WindowDriver`: private per-window driver state (its `Ui` recorder, a per-window `Frontend` encode/compose scratch, the persistent `Backbuffer`, and frame-scheduling/occlusion state) that drives the **one shared** `WgpuBackend` passed into each frame. `Ui` solely owns that window's `RecordStore`; later phases borrow it through the `Ui`, keeping each retained tree paired with its variable-length geometry. `WindowFrameInput` carries host facts into a frame and `WindowFrameOutput` returns scheduling, cursor, close resolution, and drained window commands without host reach-through. `WindowDriver` and `FramePresent` stay behind the supported `WinitHost` and `OffscreenHost` facades; both route their `App` through the same `Ui::frame` lifecycle. `OffscreenHost` retains one `HostShared`, one `WgpuBackend`, and one `WindowDriver`. The backend owns device/queue, pipelines, GPU atlases, and image textures; it receives only `BackendShared` plus `BackendConfig`, while each submission borrows the active window's `RecordPayloads`.
- `src/host/winit/` тАФ `WinitHost<T>`: winit `ApplicationHandler` glue with one `HostPhase<T>` lifecycle enum. `Bootstrap<T>` owns pre-`resumed` inputs and queued main-thread tasks; `WinitRuntime<T>` owns the app, retained `HostShared`, `SurfaceFactory`, one `WgpuBackend`, all live `WindowDriver`s, and pending window commands. `SurfaceFactory` retains the instance/adapter and surface policy; the backend is the sole device/queue owner. Events route by `WindowId`, each window's `FramePresent` folds into one `ControlFlow`, and input crosses through `WindowDriver`. `handle.rs` is `HostHandle<T>`, the cloneable off-thread handle (`request_repaint(WindowToken)`, `run_on_main`, `quit`); `config.rs` is `WinitHostConfig` (title, initial/min logical size, present mode, power preference); `gpu.rs` selects the sRGB swapchain. The swapchain is always double-buffered (`desired_maximum_frame_latency: 1`).
- `src/debug_overlay.rs` тАФ `DebugOverlayConfig` on `Ui` (damage-rect / clear-damage / frame-stats visualizations)
- `src/bin/showcase/` тАФ the showcase binary (`main.rs`, uses `WinitHost`) + multi-page demo content (~20 tabs, several features grouped per tab; shared page/section/cell scaffolding + swatch palette in `support.rs`)
- `examples/` тАФ `dump_theme` (theme TOML round-trip), `counter`, `frame_visual`, `custom_widget` (a `Stepper` authored from the public API тАФ the widget-authoring surface's compile-time proof)
- `tests/alloc/` тАФ per-frame allocation audit suite (a `CountingAllocator` global allocator + shared fixtures/harness; see `tests/alloc/alloc-testing.md`); integration-level sibling to the `alloc_free` bench, run via `cargo test --test alloc --features internals` (reaches the `internals`-gated `Ui::default`)
- `tests/visual/` тАФ headless wgpu тЖТ PNG тЖТ golden-diff suite (`cargo test --test visual --features internals`); the canonical eyeball-replacement for rendering changes. Golden PNGs are gitignored and per-machine (auto-created on first run); after an intentional render change, regenerate with `UPDATE_GOLDEN=1 cargo test --test visual --features internals <filter>` and inspect the diff artifacts under `tests/visual/output/<name>/` first. Full reference: `tests/visual/visual-testing.md`
- `src/bench/` тАФ all Criterion and `dhat` implementations plus shared workloads, arranged to mirror their production subsystem (`allocation`, `layout/cache`, `renderer/backend/{curve,text}`, `renderer/frontend/composer`, `ui/{cascade,damage}`, and so on); compiled only by `internals`
- `benches/` тАФ thin Criterion target wiring plus the three `dhat` global-allocator shells (alloc_free, alloc_free_gpu, alloc_resize, caches, cascade, composer, curve_pipeline, damage, frame, input_throughput, text_atlas, text_shape). Every target requires `--features internals`. `docs/` holds in-flight notes and media; durable subsystem rationale lives in colocated module docs, while the **Architecture** section above is the crate-wide source of truth

Key deps: `wgpu`+`winit`, `cosmic-text` (the wgpu text rendering backend lives in-tree at `src/renderer/backend/text/`), `glam`, `rustc-hash`, `bytemuck`, `soa-rs` (per-node SoA storage on `Tree`). Pinned to caret versions (lockfile is source of truth).

## Before reporting work as done

Always run, in this order, before confirming any code change:

```sh
cargo fmt --all
cargo clippy --all-targets -- -D warnings
cargo test
```

For changes that touch feature-gated code (anything under
`#[cfg(feature = ...)]`, exposed via `support::internals`, or that
might be affected by `internals`), run the full feature
matrix instead:

```sh
scripts/test-all.sh       # fmt + strict docs + clippy + tests across all feature combos
FAST=1 scripts/test-all.sh # skip fmt + docs + clippy, just run tests per combo
```

For changes that touch **rendering** (shaders, encoder/composer, gradient
or text atlas, colour pipeline, layout that moves pixels), also run the
visual suite тАФ `cargo test` alone won't catch a render regression:

```sh
cargo test --test visual --features internals
```

If goldens legitimately move (an intentional visual change), inspect the
`tests/visual/output/<name>/{actual,expected,diff}.png` artifacts, then
regenerate with `UPDATE_GOLDEN=1` and re-run to confirm green.

Fix anything that fails. Don't tell the user a change is complete unless these all pass.

## Hot-path struct sizes

`src/lib.rs`'s `hot_struct_sizes` module drives two tests from one
`hot_structs!` inventory of every per-frame struct touched by layout /
cascade / encode / compose / damage (the SoA columns, the per-shape /
per-chrome lowered forms, the encoderтЖФcomposer wire payloads, and the
GPU instance types):

- **`hot_struct_sizes_are_pinned`** тАФ a real (non-ignored) gate that
  asserts `(size, align)` for every entry. A silent footprint
  regression (added field, stop-cap bump, an enum variant re-inlining a
  boxed payload) fails `cargo test` instead of diffusing through the
  codebase.
- **`print_hot_struct_sizes`** (`#[ignore]`) тАФ prints the live table.
  Run it to read off a new number when a layout change is intentional:

```sh
cargo test --lib print_hot_struct_sizes -- --nocapture --ignored
```

When a hot row (`NodeRecord`, `LayoutCore`, `ShapeRecord`, `Brush`,
`DrawRectPayload`, `CascadeInputHash`, `DamageRegion`, `Quad`, etc.)
changes size on purpose, update the `expected_size / expected_align`
next to that type in the `hot_structs!` list тАФ that one-line edit is
the review signal. Adding a new per-frame struct? Add it to the list so
it gets both the printout and the gate.
