# Aperture architecture review

## Executive summary

Aperture's record → measure → arrange → cascade → encode/compose → paint pipeline is coherent and deliberately optimized for stable-frame reuse. The review did not find a reason for a broad rewrite or for replacing the core rendering and text dependencies. The most valuable simplifications are at the seams: several APIs represent invalid combinations with parallel fields, sentinel values, debug-only assertions, or setters that are silently discarded; persistent owners also retain transient layout context or state belonging to widgets and UI orchestration.

The highest-priority work is therefore to make layout, widget, shape, image, and text-shaping contracts explicit. The next tier moves state to its natural owner and deletes redundant retained data. The structural batches then split the few load-bearing files whose types currently span multiple independently changing responsibilities. Dependency reduction is worthwhile but focused: remove `fixedbitset` and `strum`, feature-gate `pollster`, and trim the proc-macro parser feature instead of disturbing purposeful dependencies such as `wgpu`, `cosmic-text`, `etagere`, or `unicode-segmentation`.

Current flow: `Ui::frame` records `Forest`/`RecordStore`; `LayoutEngine` measures and arranges it; `CascadesEngine` derives paint, clip, hit-test, and damage inputs; the renderer frontend encodes and composes commands; `WgpuBackend` uploads resources and submits passes. `InputState`, `TextShaper`, image resources, widget state, and scroll state persist beside that pipeline. Most findings below tighten ownership between those side stores and the pass that actually uses them.

## Batch 1 — High: Make invalid API and data states unrepresentable

- [ ] **Replace parallel `LayoutMode` and `ModePayload` fields with a discriminated layout kind.** `ModePayload` can only recover a grid ID or scroll spec when the separately supplied mode matches, and enforces that relationship with `debug_assert_eq!` (`aperture/src/layout/types/layout_mode.rs:71-105`). Both authoring `Element` and hot `LayoutCore` store the pair independently (`aperture/src/scene/element/mod.rs:111-115`, `aperture/src/scene/element/columns.rs:168-190`), while a grid is deliberately constructed as `Grid + NONE` and patched later (`aperture/src/scene/element/mod.rs:204-225`). Introduce an authoring `LayoutKind::{Leaf, HStack, ..., Grid(GridDefId), Scroll(ScrollSpec)}` and a compact typed representation in `LayoutCore`; construct a grid with its arena ID in the same operation that records its definition. Preserve the pinned hot-struct size with private packing only if measurement shows it matters. Validate every driver dispatch and hash, grid/scroll recording, and the `LayoutCore` size guard.

- [ ] **Replace the universal `Configure` contract with capability-specific configuration.** `Configure` promises all element setters to every implementer (`aperture/src/scene/element/mod.rs:311-390`), but `ContextMenu` rejects caller identity only with a debug assertion before overwriting it (`aperture/src/widgets/context_menu/mod.rs:125-149`, `aperture/src/widgets/context_menu/mod.rs:187-191`), `Tooltip` likewise owns and overwrites identity (`aperture/src/widgets/tooltip/mod.rs:182-211`), and `Scroll` discards every caller transform because pan/zoom owns the field (`aperture/src/widgets/scroll/mod.rs:345-386`). Split identity, sizing/placement, container layout, clipping, interaction, and transform into narrow traits and implement only the supported capabilities; rename `Spinner::size(f32)` to `diameter` to stop shadowing layout sizing (`aperture/src/widgets/spinner.rs:56-60`). Validate the supported setter matrix and update internal consumers so unsupported combinations no longer compile instead of panicking in debug or doing nothing in release.

- [ ] **Replace theme and builder sentinels with explicit override state.** `Spacing::ZERO` means both a real zero and "inherit" in `resolve_look` (`aperture/src/widgets/theme/mod.rs:207-227`), and the public theme docs acknowledge that callers cannot request zero padding (`aperture/src/widgets/theme/button.rs:18-22`, `aperture/src/widgets/theme/text_edit.rs:22-26`). `ClipMode::None` and `Option<Background>::None` similarly mean inherit in container chrome (`aperture/src/widgets/mod.rs:37-54`), while `Sizes::default()` means "not configured" in Slider, Separator, and ProgressBar (`aperture/src/widgets/slider.rs:94-100`, `aperture/src/widgets/separator.rs:62-75`, `aperture/src/widgets/progress_bar.rs:40-51`). Keep the compact, resolved `Element` concrete, but retain `Inherit | Value(T)` metadata on relevant builders; optional chrome needs `Inherit | Value(Option<T>)`. Validate explicit zero padding/margin, Hug×Hug sizing, no background, and `ClipMode::None` against non-default themes.

- [ ] **Return shape-specific builders instead of exposing variant-dependent methods on `Shape`.** Constructors return the broad enum, then `fill`, `stroke`, `corners`, `at`, `brush`, `cap`, `join`, and image setters match over variants and panic on unsupported combinations (`aperture/src/shape/mod.rs:239-382`, `aperture/src/shape/mod.rs:384-516`); even a gradient passed to Triangle's otherwise available `fill` panics at runtime (`aperture/src/shape/mod.rs:386-395`). Introduce typed payload/builders such as `RectShape`, `TriangleShape`, `CurveShape`, `PolylineShape`, `ImageShape`, and `ShadowShape`, each exposing only valid setters, then convert them into the internal `Shape` at `Ui::add_shape`. This removes repeated variant matches from the public API and moves misuse to compile time. Validate every constructor/setter combination and exact lowered `ShapeRecord` output.

- [ ] **Remove nested `Deref` from composite widget responses.** `Response` already uses `Deref` as the deliberate lazy projection to `ResponseState` (`aperture/src/widgets/mod.rs:109-173`), but `InnerResponse<R>` adds another `Deref` to `Response` and explicitly cannot grow methods or fields without shadowing that target (`aperture/src/widgets/mod.rs:204-225`). `DragValueResponse` and `TextEditResponse` repeat the same outer projection (`aperture/src/widgets/drag_value/mod.rs:150-176`, `aperture/src/widgets/text_edit/mod.rs:801-829`). Keep the single `Response -> ResponseState` convenience, rename `InnerResponse<R>` to `ContainerResponse<R>`, and require composite results to expose interaction through an explicit `.response`; this preserves lazy input resolution while making ownership and method lookup obvious. Validate Panel, Grid, Scroll, DragValue, and TextEdit call sites, including showcase and Darkroom consumers, and assert response resolution counts do not increase.

## Batch 2 — High: Put transient and cross-frame state under the right owner

- [ ] **Reject unsupported image dimensions when registering, before queuing GPU work.** `ImageHandle` stores `U16Vec2` metadata (`aperture/src/renderer/image_registry.rs:51-55`), registration is infallible (`aperture/src/renderer/image_registry.rs:136-146`, `aperture/src/ui/mod.rs:726-733`), and `u16_size` silently saturates (`aperture/src/renderer/image_registry.rs:176-182`); the real device limit is checked only during backend upload, where it panics (`aperture/src/renderer/backend/image_pipeline/textures.rs:35-44`, `aperture/src/renderer/backend/image_pipeline/textures.rs:81-89`). Retain the true `UVec2`, reject the shape-record representation limit explicitly, and make `register_image` return `Result<ImageHandle, RegisterImageError>` after checking the selected device limit. Rejected images must never enter `pending`. Validate both exact limits, one-pixel overflow on each axis, pending-queue behavior, and maximum accepted size through shape lowering.

- [ ] **Remove `LayoutEngine::active_layer` and thread the current `&mut LayerLayout` through recursive layout calls.** `active_layer` is transient ambient context retained on a persistent engine (`aperture/src/layout/engine.rs:109-141`); `run` sets it per layer, while measure, arrange, cache restoration, text shaping, and subtree clearing accept the entire multi-layer output and index through that hidden field (`aperture/src/layout/engine.rs:486-595`, `aperture/src/layout/engine.rs:609-700`, `aperture/src/layout/engine.rs:786-847`, `aperture/src/layout/support.rs:219-233`). Borrow one layer output at the orchestration boundary and pass it through drivers. This narrows mutation to the only layer a driver can touch and prevents helpers from silently targeting whichever layer happened to run last. Validate every driver, cache-hit restoration, container text, and multi-layer popup/modal layout; confirm the change stays allocation-neutral.

- [ ] **Move scroll ownership to `Ui` and separate interaction state from layout metrics.** `ScrollLayoutState` interleaves layout output (`viewport`, `outer`, `content`, `overflow`, `seen`) with interaction state (`offset`, `zoom`, `content_margin`, `drag_anchor`) and implements wheel, zoom, drag, and page behavior inside the layout module (`aperture/src/layout/scroll/mod.rs:36-95`, `aperture/src/layout/scroll/mod.rs:127-300`). Widgets consequently mutate `layout_engine.scroll_states` directly (`aperture/src/widgets/scroll/mod.rs:619-705`), making a persistent layout engine own widget input state (`aperture/src/layout/engine.rs:129-141`). Introduce a UI-owned `ScrollStore` whose row contains explicit `ScrollInteraction` and `ScrollMetrics`; widgets mutate interaction, layout reads interaction and writes metrics, and sweeping remains centralized. Validate cold mount, cache-hit content retention, wheel/thumb/pivot zoom, removal, and steady-state allocations.

- [ ] **Make bounded text shaping consume the fresh unbounded result instead of relying on temporal call order.** `shape_wrap` requires a same-frame `shape_unbounded` for the same identity (`aperture/src/text/mod.rs:373-390`), panics if the entry is absent, and checks stale authoring only with `debug_assert!` (`aperture/src/text/mod.rs:402-418`); release code can therefore reuse an old unbounded key if a caller violates the convention. The layout engine currently obeys the order manually (`aperture/src/layout/engine.rs:869-889`, `aperture/src/layout/engine.rs:930-946`). Return an opaque `UnboundedReuse` token containing the validated identity, key, and measurement from `shape_unbounded`, require it in `shape_wrap`, and remove the second identity lookup. Validate cache hits/misses, authoring changes, wrap-target changes, and compile-time impossibility of stale or missing-parent calls.

- [ ] **Make TextEdit's model semantic and keep host input, clipboard, paint, and menu policy at its edges.** `TextEditState` combines cursor/selection, retained paint scratch, focus/blink, undo/redo, and viewport scrolling (`aperture/src/widgets/text_edit/model.rs:22-69`), while the nominal model imports Clipboard, platform detection, keys, and shortcuts and performs clipboard/shortcut dispatch itself (`aperture/src/widgets/text_edit/model.rs:3-7`, `aperture/src/widgets/text_edit/model.rs:239-269`, `aperture/src/widgets/text_edit/model.rs:308-358`). `TextEdit::show` then orchestrates five phases and menu behavior in one method (`aperture/src/widgets/text_edit/mod.rs:263-285`, `aperture/src/widgets/text_edit/mod.rs:429-725`). Keep one retained row for lookup efficiency but compose it from cursor, history, viewport, and paint-scratch state; have `input.rs` translate platform events into semantic edit/navigation commands, `paint.rs` emit shapes, and `context_menu.rs` own default menu policy. Validate the existing editing behavior, clipboard failures, focus/blink, viewport movement, visuals, and allocation behavior.

## Batch 3 — Medium-high: Delete redundant and parallel retained state

- [ ] **Move GPU-agnostic image lifecycle and texture identity out of `renderer`.** `UiResources` imports both `ImageRegistry` and `TextureIdSource` from renderer (`aperture/src/ui/resources.rs:1-17`), the public `Shape` layer imports `ImageHandle` from renderer (`aperture/src/shape/mod.rs:4-20`), and scene records import `TextureId` from renderer (`aperture/src/scene/shapes/record.rs:1-15`). Yet `ImageRegistry` itself is CPU-side RAII ownership plus pending/drop queues and contains no `wgpu` type (`aperture/src/renderer/image_registry.rs:22-25`, `aperture/src/renderer/image_registry.rs:41-119`, `aperture/src/renderer/image_registry.rs:122-173`). Create a top-level `image` or `resources/image` module owning `ImageHandle`, `ImageRegistry`, `TextureId`, and `TextureIdSource`; keep upload/bind-group consumption in the backend. This removes lower-level authoring/UI modules' conceptual dependency on the renderer and gives both registered images and `GpuView` one neutral ID authority. Validate shared ID uniqueness, RAII release, one-shot upload, and public API paths.

- [ ] **Delete `ScrollStates` from `cascade_fingerprint`.** The fingerprint scans every scroll row for offset and zoom (`aperture/src/scene/cascade/mod.rs:574-646`), but `Scroll::show` has already copied those values into the viewport `Element.transform` (`aperture/src/widgets/scroll/mod.rs:747-800`); `PanelExtras::hash` includes that transform (`aperture/src/scene/element/columns.rs:98-113`), and the root subtree hash is already folded into the fingerprint (`aperture/src/scene/tree/mod.rs:191-205`, `aperture/src/scene/cascade/mod.rs:601-635`). Remove the map parameter and XOR fold. Besides deleting a full-map scan and a cross-subsystem dependency, this avoids an extra cascade miss from removed scroll rows because fingerprinting precedes state sweeping (`aperture/src/ui/mod.rs:414-442`). Validate offset/zoom changes through the transform hash and stable behavior on scroll removal.

- [ ] **Replace `DragAnchor` and `DragEdit` rows with one explicit `DragValueState`.** Scrub and edit modes currently occupy separate type-erased rows with `armed` and `editing` latches (`aperture/src/widgets/drag_value/mod.rs:120-148`), and mutual exclusion is maintained through scattered writes during draft resolution, scrubbing, release, and edit entry (`aperture/src/widgets/drag_value/mod.rs:303-364`, `aperture/src/widgets/drag_value/mod.rs:409-447`). Store a reusable `String` beside `DragMode::{Idle, Scrubbing { base, speed, last }, Editing}` in one row. This makes simultaneous armed/editing states unrepresentable, retains the buffer allocation, and removes duplicate map probes for one widget ID. Validate click-to-edit, programmatic focus, every scrub edge, disable/editable changes mid-gesture, Escape, Enter, and click-away.

- [ ] **Unify rounded and windowed rectangles with an explicit rect paint mode.** `Shape::RoundedRect` and `Shape::WindowedRect` duplicate the same four fields (`aperture/src/shape/mod.rs:33-58`), and the duplication continues into `ShapeRecord` (`aperture/src/scene/shapes/record.rs:45-66`, `aperture/src/scene/shapes/record.rs:229-243`), lowering (`aperture/src/scene/shapes/mod.rs:76-106`), hashing (`aperture/src/scene/shapes/hash.rs:23-48`), and encoding (`aperture/src/renderer/frontend/encoder/mod.rs:309-331`). Use one rectangle payload with `RectPaint::{Interior, ExteriorWindow}` (or equivalent), include the mode in the record hash/tag, and branch once when emitting the draw command. This removes parallel variants without erasing the shader's window flag. Validate identical hashes for identical mode/payload, distinct hashes across modes, ordinary/window visual output, damage, and occlusion behavior.

- [ ] **Remove `fixedbitset` by deriving grid resolution and storing text-owning containers sparsely.** The dependency has only two production uses (`aperture/Cargo.toml:29`): `AxisScratch.resolved` parallels track sizes and `commit_fill` state (`aperture/src/layout/grid/mod.rs:50-83`, `aperture/src/layout/grid/mod.rs:432-503`, `aperture/src/layout/grid/mod.rs:709`, `aperture/src/layout/grid/mod.rs:783-818`), while `container_text` is populated during one reverse tree traversal and iterated once later (`aperture/src/scene/tree/rollups.rs:31-52`, `aperture/src/scene/tree/mod.rs:245-267`, `aperture/src/layout/engine.rs:577-595`). Replace the grid bitset with an explicit resolution phase (`FixedOnly`, `FixedAndHug`, `All`) and test spanned track kinds directly; replace container owners with a retained `Vec<u32>` consumed in the required order. Validate every fixed/hug/fill span combination, bounded/unbounded grids, wrapped container text, nested grids, and compare dense versus sparse scenes before deleting the crate.

## Batch 4 — Medium: Split load-bearing modules along existing responsibilities

- [ ] **Compose `InputState` from pointer, keyboard/focus, and frame-wake domains.** The flat structure owns routing/capture/scroll queues, keyboard/focus/modifiers, subscriptions, action/repaint flags, quiescence, and the frame clock (`aperture/src/input/mod.rs:258-358`); its event dispatcher spans `aperture/src/input/mod.rs:446-680`, and response projection is another substantial responsibility (`aperture/src/input/mod.rs:799-925`). Keep `InputState` as the façade but nest `PointerState`, `KeyboardState`, and `FrameWakeState`, placing event handlers in their canonical modules and letting response projection delegate to pointer state before adding focus. During the split, validate all vector input at ingress: zoom rejects invalid factors (`aperture/src/input/mod.rs:452-457`), but pointer positions and scroll deltas accept non-finite values into persistent capture/delta math (`aperture/src/input/mod.rs:465-495`, `aperture/src/input/mod.rs:594-617`). Validate pointer capture, focus, subscriptions, wake policy, and malformed-event transition tables.

- [ ] **Split `text/mod.rs` into public metrics, shaping, reuse, editing geometry, and input-handle modules.** The root file combines public font/config/measurement types, shared shaper/cache state, identity reuse, renderer bridging, and cursor/hit/selection geometry (`aperture/src/text/mod.rs:61-274`, `aperture/src/text/mod.rs:276-445`, `aperture/src/text/mod.rs:447-758`, `aperture/src/text/mod.rs:858-1028`). Its imports consequently mix layout alignment, `WidgetId`, `cosmic_text`, cache maps, and Unicode editing (`aperture/src/text/mod.rs:24-35`). Move metrics/config/key types to `metrics.rs`, shaping to `shaper.rs`, retained identity reuse to `reuse.rs`, and cursor/selection geometry to `editing.rs`. Move `TextInput`/`InternedStr` into `text/input.rs`, while scene-only `RecordedText`/`ResolvedText` leave `primitives/interned_str.rs` (`aperture/src/primitives/interned_str.rs:7-61`, `aperture/src/scene/record_store.rs:142-210`). Keep published re-exports only in `lib.rs`. Validate public API compilation, layout reuse, editing, and renderer reconstruction.

- [ ] **Split the cascade module into state/query, traversal engine, and paint geometry.** The 1,227-line module combines frozen SoA data and hit testing (`aperture/src/scene/cascade/mod.rs:37-442`), update/rebuild policy and traversal (`aperture/src/scene/cascade/mod.rs:444-872`), and shape/text/stroke paint-bound computation (`aperture/src/scene/cascade/mod.rs:874-1224`). Create `cascade/state.rs`, `cascade/engine.rs`, and `cascade/paint.rs`, keeping each struct with its inherent implementation and updating callers to canonical submodule paths rather than adding internal re-exports. This separates the stable query surface from dependencies that change with rendering primitives. Validate incremental/full cascade equivalence, hit targeting, damage rows, text/stroke bounds, and visual fixtures.

- [ ] **Separate structural scene diffing from damage-region accumulation.** `DamageEngine` owns the damage budget and raw region, retained scene snapshots and paint arena, and structural traversal scratch in one struct (`aperture/src/scene/damage/mod.rs:73-147`). Its `compute` method performs the retained-tree diff (`aperture/src/scene/damage/mod.rs:292-713`), while paint-only invalidation is a separate path (`aperture/src/scene/damage/mod.rs:728-742`) and geometry/order helpers continue through `aperture/src/scene/damage/mod.rs:750-900`. Move retained snapshots, `PaintSnapArena`, ordering extents, parent stack, and the structural walk into `damage/diff.rs` under a `StructuralDamage` owner; let it handle add/remove/reparent/move/order changes plus snapshot eviction and compaction. Keep budget, raw rectangles, and region collapse in `DamageEngine`, which coordinates structural results, animation pre-damage, and paint-only invalidation. Validate exact `Damage` output for add, remove, reparent, move, order inversion, paint-only, and full-damage cases, then compare the damage benchmark and steady-state allocations.

## Batch 5 — Medium: Isolate renderer and widget state machines

- [ ] **Move batch and polyline state machines out of `Composer::compose`.** `Composer` owns many independent state domains (`aperture/src/renderer/frontend/composer/mod.rs:60-93`); `BatchState` is declared separately but its open/close/hit/text lifecycle is spread across composer methods (`aperture/src/renderer/frontend/composer/mod.rs:121-129`, `aperture/src/renderer/frontend/composer/mod.rs:195-322`, `aperture/src/renderer/frontend/composer/mod.rs:387-429`, `aperture/src/renderer/frontend/composer/mod.rs:1141-1212`), while polyline transformation and emission occupy a large inline command arm (`aperture/src/renderer/frontend/composer/mod.rs:964-1140`). Add `composer/batch.rs` with `BatchState`/`OpenBatch` behavior and `composer/polyline.rs` with retained `PolylineScratch` preparation/emission; leave `compose` responsible for clip, transform, and command order. Validate exact `RenderBuffer` equality, overlap/order cases, polyline degeneracy and joins, and retained allocation counts.

- [ ] **Split command traversal from shape and image lowering in the encoder.** `Encoder` and `GradientResolver` share one module (`aperture/src/renderer/frontend/encoder/mod.rs:42-85`); the file then combines tree traversal and command dispatch (`aperture/src/renderer/frontend/encoder/mod.rs:128-227`), collision diagnostics (`aperture/src/renderer/frontend/encoder/mod.rs:248-284`), shape lowering (`aperture/src/renderer/frontend/encoder/mod.rs:286-720`), shadow emission (`aperture/src/renderer/frontend/encoder/mod.rs:722-771`), and image-fit geometry (`aperture/src/renderer/frontend/encoder/mod.rs:773-862`). Move `GradientResolver` and shape/shadow lowering to `encoder/shape.rs`, and image geometry to `encoder/image_fit.rs`; keep `Encoder` with traversal, clipping, text dispatch, and collision orchestration in `mod.rs`. Keep inherent implementations beside their structs and use canonical paths without internal re-exports. Validate exact `RenderCmdBuffer` output for every shape variant, image fit/alignment mode, text, clipping, and collision-overlay state.

- [ ] **Split scroll orchestration from zoom and scrollbar mechanics, and group per-axis state.** The widget module combines zoom configuration (`aperture/src/widgets/scroll/mod.rs:23-91`), scrollbar geometry, interaction, planning, and rendering (`aperture/src/widgets/scroll/mod.rs:101-343`), configuration forwarding (`aperture/src/widgets/scroll/mod.rs:345-412`), and a `show` method that coordinates routing, wheel/zoom input, thumb/page interaction, layout state, and scene recording (`aperture/src/widgets/scroll/mod.rs:414-853`). `BarResponses` and `BarPlans` also store the same vertical/horizontal axes in parallel structures (`aperture/src/widgets/scroll/mod.rs:192-229`). After introducing `ScrollStore`, create `scroll/zoom.rs` and `scroll/bar.rs`; let the bar module own geometry, interaction planning, and rendering, and represent each axis with one `BarAxisFrame` rather than parallel response/plan fields. Keep `Scroll` and `show` orchestration in `mod.rs`. Validate every existing wheel, zoom, thumb, track-click, overflow, axis-enable, and overlay case and confirm warm-frame allocation behavior is unchanged.

- [ ] **Extract target allocation and frame submission from `WgpuBackend`.** The backend owns device/queue/belt, all drawing systems, format caches, images, diagnostics, and timing (`aperture/src/renderer/backend/mod.rs:129-175`), then also creates resources, allocates per-window targets, performs upload/pass/copy/submit, and translates schedules into render-pass calls (`aperture/src/renderer/backend/mod.rs:178-240`, `aperture/src/renderer/backend/mod.rs:271-336`, `aperture/src/renderer/backend/mod.rs:382-647`, `aperture/src/renderer/backend/mod.rs:773-966`). Move `Backbuffer`, `Stencil`, and allocation functions to `backend/targets.rs`; introduce a borrowing `FrameSubmitter<'_>` in `backend/submit.rs` for upload → pass → copy → queue orchestration; keep `WgpuBackend` as the resource composition root and façade. Validate full/partial/skip schedules, direct/backbuffer presentation, resize/format changes, stencil, overlays, GPU views, and timing-enabled paths.

## Batch 6 — Low: Reduce dependency and hot-path surface

- [ ] **Move GPU wire vocabulary out of general-purpose primitives.** `primitives/fill_wire.rs` explicitly defines shader tags, packed flags, and a gradient-atlas row (`aperture/src/primitives/fill_wire.rs:1-30`, `aperture/src/primitives/fill_wire.rs:77-136`), while `FillAxis` is another GPU-wire type embedded in the public brush module (`aperture/src/primitives/brush/mod.rs:14-59`). Create a private top-level `paint_wire` module shared by scene lowering and renderer; keep authoring `Brush`/gradient types in `primitives::brush`, and move atlas-specific `LutRow` into `renderer::gradient_atlas`. This preserves downward dependency direction without presenting shader ABI as a geometric primitive. Validate Pod sizes/offsets, WGSL constant parity, gradient/shadow/triangle/window lowering, and render hashes.

- [ ] **Tighten three small dependency declarations.** `pollster` is unconditional (`aperture/Cargo.toml:36`) although production use is confined to winit GPU bootstrap and the `internals` GPU harness (`aperture/src/host/winit/gpu.rs:62-97`, `aperture/src/host/test_gpu.rs:9-36`); make it optional and add it to `winit-host` and `internals` (`aperture/Cargo.toml:165-177`). `strum` plus derive macros supports only four fixed enums (`aperture/Cargo.toml:41`, `aperture/src/input/pointer.rs:13-29`, `aperture/src/scene/layer.rs:4-18`, `aperture/src/diagnostics/gpu_stats.rs:22-80`, `aperture/src/renderer/frontend/cmd_buffer/payload.rs:16`, `aperture/src/renderer/frontend/cmd_buffer/payload.rs:125-127`); replace it with local `COUNT`/`ALL` constants and explicit `from_u8`/label matches. Finally, the derive crate parses `DeriveInput` and named fields but enables all of Syn (`aperture/anim-derive/Cargo.toml:17-20`); drop `full` and enable only the parser features compilation requires. Validate default, no-default-feature, `winit-host`, `internals`, and standalone `anim-derive` builds.

- [ ] **Gate text measurement counters out of shipping builds.** `measure_calls` is explicitly test instrumentation but lives in production `ShaperInner` (`aperture/src/text/mod.rs:105-108`, `aperture/src/text/mod.rs:157-165`) and is incremented on every shaping dispatch (`aperture/src/text/mod.rs:308-319`, `aperture/src/text/mod.rs:323-348`, `aperture/src/text/mod.rs:382-420`). Its only reader is gated support (`aperture/src/text/mod.rs:1071-1080`). Put the field and increments behind `#[cfg(any(test, feature = "internals"))]`, preferably through one cfg-only helper. Validate default, release, and `internals` builds plus cache-effectiveness instrumentation.

- [ ] **Use the existing record-arena formatter in the frame-stats overlay.** The opt-in overlay allocates one or two heap `String`s per displayed frame and then copies the result into the frame arena (`aperture/src/ui/frame_stats.rs:18-28`, `aperture/src/ui/frame_stats.rs:47`), although `Ui::fmt` exists to write formatting arguments directly into that arena (`aperture/src/ui/mod.rs:777-790`). Branch on optional GPU timing and pass `format_args!` directly to `ui.fmt`. Validate identical labels with and without GPU timing and run the allocation audit with frame stats enabled.

## Open questions

- [ ] **Is the 28-byte `LayoutCore` size a measured cache-density requirement or only a regression guard?** The layout-kind change should preserve it unless a focused benchmark shows that a slightly larger natural enum improves maintainability without meaningful traversal cost (`aperture/src/scene/element/columns.rs:168-190`).

- [ ] **Can image registration happen before the host has selected a device?** The answer determines whether the maximum texture dimension belongs in `ImageRegistry` construction or is installed once after GPU initialization; either way, the current silent `u16` saturation must be removed (`aperture/src/renderer/image_registry.rs:122-146`, `aperture/src/renderer/image_registry.rs:176-182`).

- [ ] **Are direct-text container owners dense in real scenes?** If they can be dense, benchmark the proposed retained sparse vector against a standard-library packed representation before removing `fixedbitset` (`aperture/src/scene/tree/rollups.rs:31-52`, `aperture/src/layout/engine.rs:577-595`).
