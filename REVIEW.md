# Aperture codebase review

## Executive summary

The highest-impact risks are at public host boundaries. Normal platform failures during Winit and GPU initialization abort the process, invalid offscreen scale factors reach release rendering with only a debug assertion, and non-finite pointer or scroll input can enter retained interaction state. These paths turn recoverable environment or input failures into crashes or persistent invalid geometry.

The main simplification opportunities are narrower but repeated across hot subsystems. Text validation loses its proof between recording and layout, editor-only geometry occupies the shared shaping/cache module, DragValue represents one interaction as two independently stored latches, and rounded/windowed rectangles carry parallel retained payloads through lowering and encoding.

Release hot paths also retain invariant checks and test instrumentation, while the opt-in frame overlay allocates strings before copying them into the existing text arena. Lower-priority findings cover duplicated widget/input vocabulary, unused public surface, overly broad dependency features, and architecture documentation that no longer matches the storage layout.

## High: public boundaries can crash or poison retained state

- [ ] **Expected Winit and GPU startup failures panic through the public host API.** `WinitHostBuilder::build` returns `WinitHost` directly but unwraps event-loop creation, and `WinitHost::run` unwraps the event-loop result (`src/host/winit/mod.rs:349-381`, `src/host/winit/mod.rs:408-412`). Runtime bootstrap and app-requested window creation likewise return no failure channel (`src/host/winit/mod.rs:177-220`, `src/host/winit/mod.rs:236-253`), while window, surface, adapter, device, and surface-format selection all use `expect` (`src/host/winit/mod.rs:502-525`, `src/host/winit/gpu.rs:46-99`, `src/host/winit/gpu.rs:125-133`, `src/host/winit/gpu.rs:144-175`). Missing adapters, device-request failures, unsupported displays, and ordinary OS window-creation failures therefore terminate the application instead of remaining expected host errors.

- [ ] **Offscreen display scale validation disappears in release builds.** `OffscreenHost::frame_offscreen` accepts an arbitrary `f32` scale factor and passes it directly into `Display::from_physical` (`src/host/offscreen.rs:151-172`, `src/host/offscreen.rs:182-204`); `Display` stores the value unchanged and divides physical dimensions by it (`src/display.rs:56-74`). The only frame-boundary guard is a `debug_assert!` that does not reject positive infinity even in debug and is removed entirely in release (`src/ui/mod.rs:163-184`). Zero, negative, infinite, or NaN caller input can consequently feed infinite or NaN geometry into layout, damage, composition, and raster scaling.

- [ ] **Pointer and scroll events accept non-finite coordinates into persistent interaction state.** The public `InputEvent` documents and carries raw `Vec2` values for pointer and both scroll variants (`src/input/mod.rs:154-185`), but ingress rejects only invalid `Zoom` factors (`src/input/mod.rs:527-550`). Pointer positions are retained and used for drag-distance and hit-test calculations, while scroll vectors are accumulated without validation (`src/input/mod.rs:546-575`, `src/input/mod.rs:675-697`). A non-finite scroll component then survives the widget boundary and is written into retained offsets because `NaN != 0.0` and clamping preserves NaN (`src/widgets/scroll/mod.rs:561-591`, `src/widgets/scroll/state.rs:106-128`). One malformed event can therefore strand hover/capture routing or permanently poison a scroll viewport.

## Medium: one responsibility is represented in multiple places

- [ ] **Validated text metrics lose their proof between recording and layout.** Shape recording drops text whose font size or line height is invalid before lowering (`src/shape/mod.rs:300-320`, `src/scene/shapes/mod.rs:71-77`), but the retained record stores the same metrics as raw `f32` fields (`src/scene/shapes/record.rs:87-125`, `src/scene/shapes/mod.rs:148-170`). Layout reconstructs a fallible `TextShapeRequest` and immediately `expect`s success for every run (`src/layout/support.rs:23-54`), on both desired-size and intrinsic-size paths (`src/layout/engine.rs:837-864`, `src/layout/intrinsic/mod.rs:251-279`). The trusted hot path repeats finite/epsilon checks and carries an impossible failure branch because the earlier validation result has no representation.

- [ ] **Editor-only geometry occupies the shared shaping/cache root module.** `TextLayoutProbe`, `SelectionRects`, cursor positioning, hit testing, grapheme selection geometry, and byte/cursor conversion all live beside public font vocabulary, shared cache ownership, and shaping dispatch (`src/text/mod.rs:24-39`, `src/text/mod.rs:62-67`, `src/text/mod.rs:165-171`, `src/text/mod.rs:496-634`, `src/text/mod.rs:1001-1093`). Their production consumers are confined to TextEdit (`src/widgets/text_edit/view.rs:1-21`, `src/widgets/text_edit/view.rs:229-239`, `src/widgets/text_edit/input.rs:106-124`). The generic text subsystem consequently owns editor interaction semantics and the `unicode-segmentation` dependency, widening a load-bearing module and its change surface.

- [ ] **DragValue stores one mutually exclusive interaction as two independent state-map rows.** Scrubbing and inline editing use separate `DragAnchor::armed` and `DragEdit::editing` latches (`src/widgets/drag_value/mod.rs:121-149`). Their exclusion depends on distributed probes and writes during draft resolution, drag start/stop, editor entry, submission, disable, and focus loss (`src/widgets/drag_value/mod.rs:273-355`, `src/widgets/drag_value/mod.rs:406-472`). Invalid combinations remain representable, and each gesture pays repeated type-erased state lookups while correctness depends on every transition updating both rows consistently.

- [ ] **Rounded and windowed rectangles duplicate one retained payload through the render pipeline.** Authoring represents both through `RectShape { kind, local_rect, corners, fill, stroke }` (`src/shape/rect.rs:7-25`), but lowering expands that into two `ShapeRecord` variants with identical fields (`src/scene/shapes/record.rs:45-66`, `src/scene/shapes/record.rs:229-243`, `src/scene/shapes/mod.rs:78-96`). Hashing explicitly merges the two field schedules and encoding splits them again only to select the final draw tag (`src/scene/shapes/hash.rs:23-48`, `src/renderer/frontend/encoder/mod.rs:304-326`). Rectangle payload changes therefore require parallel edits across retained representation, lowering, hashing, and emission.

## Medium-low: shipping hot paths retain avoidable work

- [ ] **Release assertions execute at per-node, per-command, and per-shape grains.** `quantize_available` performs a release assertion on availability values used during layout and text-width quantization (`src/layout/cache/mod.rs:62-75`, callers `src/layout/engine.rs:325-333`, `src/layout/engine.rs:589-599`). Every emitted renderer command checks the packed arena offset (`src/renderer/frontend/cmd_buffer/mod.rs:403-407`, `src/renderer/frontend/cmd_buffer/mod.rs:464-470`), and tree finalization checks paint-animation columns every record pass (`src/scene/tree/mod.rs:139-158`). Mesh construction adds another release bounds assertion per triangle (`src/primitives/mesh.rs:142-159`). Normal release frames retain invariant branches at exactly the node/command/shape frequencies the crate otherwise optimizes out.

- [ ] **Test-only text dispatch accounting mutates production shaping state.** `ShaperInner` unconditionally stores `measure_calls` and documents it as cache-effectiveness instrumentation (`src/text/mod.rs:181-196`). Direct probes, identity misses, and bounded misses increment the counter on shipping paths (`src/text/mod.rs:264-281`, `src/text/mod.rs:347-350`, `src/text/mod.rs:464-488`), while its only reader lives in the test/internals-gated support module (`src/text/mod.rs:1134-1137`, `src/text/mod.rs:1289-1302`). Every real shaping dispatch pays a shared mutable-state write for data normal builds cannot observe.

- [ ] **The frame-stats overlay heap-allocates text before copying it into the retained record arena.** Each enabled frame constructs a GPU suffix `String`, then a second label `String`, then passes that label through `Ui::intern` (`src/ui/frame_stats.rs:14-28`, `src/ui/frame_stats.rs:37-49`). The existing `Ui::fmt` path already writes `fmt::Arguments` directly into the same arena (`src/ui/mod.rs:783-796`, `src/scene/record_store.rs:191-199`). The diagnostic intended to measure frame behavior therefore adds one or two avoidable heap allocations per displayed frame.

## Low: duplicated vocabulary and unused surface increase code amount

- [ ] **`TargetDeltas` re-declares the scroll payload and manually projects it into `ScrollDelta`.** The retained accumulator duplicates the `pixels`, `lines`, and identity-valued `zoom` fields of the public result (`src/input/mod.rs:240-257`, `src/input/response.rs:188-225`). `scroll_delta_for` copies each field individually (`src/input/mod.rs:846-856`), creating two definitions and a synchronization point for one piece of routed input state.

- [ ] **Switch repeats most of the shared toggle-row implementation.** The toggle helper owns look animation, row gap, vertical alignment, indicator leaf, and label recording for Checkbox and RadioButton (`src/widgets/toggle.rs:47-98`). Switch repeats look animation, row configuration, label construction, and the Checkbox click-flip behavior around its track-specific geometry (`src/widgets/switch.rs:61-83`, `src/widgets/switch.rs:95-123`, `src/widgets/checkbox/mod.rs:58-72`). Basic toggle semantics and label layout therefore have parallel implementations that can drift.

- [ ] **Several public helpers have no non-test callers anywhere in the workspace.** The unused surface includes five directional `Corners` constructors (`src/primitives/corners.rs:49-78`), the floating-point `Color::midpoint` (`src/primitives/color.rs:95-103`), `Size::is_inf` (`src/primitives/size.rs:28-34`), and `TranslateScale::from_scale_about` (`src/primitives/transform.rs:74-89`). These APIs and their tests/documentation advertise vocabulary that the current framework does not exercise or maintain through production use.

## Low: dependency and architecture metadata have drifted

- [ ] **Dependency features are broader than the code that consumes them.** `pollster` is unconditional even though its production use is confined to the optional Winit host, with the other use in the internals-only GPU harness (`Cargo.toml:23-47`, `Cargo.toml:171-183`, `src/host/winit/gpu.rs:46-99`, `src/host/test_gpu.rs:1-37`). The derive crate enables Syn's entire `full` grammar while its implementation parses only `DeriveInput`, named struct fields, types, identifiers, and nested field metadata (`anim-derive/Cargo.toml:17-20`, `anim-derive/src/lib.rs:11-44`, `anim-derive/src/lib.rs:173-193`). Minimal and offscreen builds consequently compile dependency surface unrelated to their enabled behavior.

- [ ] **The architecture guide describes obsolete storage ownership and module paths.** Its SoA section says `BoundsExtras` holds transform/position and `PanelExtras` holds grid/scroll data (`AGENTS.md:41`), while current rows are `BoundsExtras { position, grid, min_size, max_size }` and `PanelExtras { gaps, justify, child_align, transform }` (`src/scene/node/columns.rs:71-85`). It also points Popup at the removed `src/widgets/popup.rs` path and says the toggle helper backs Checkbox/Switch (`AGENTS.md:45`, `AGENTS.md:68`), whereas Popup is a directory module and the helper explicitly backs Checkbox/RadioButton (`src/widgets/toggle.rs:47-53`). Maintainers following the crate-level source of truth are directed to the wrong ownership boundaries and files.
