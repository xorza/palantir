# Text architecture and API review

The split between window-local identity reuse (`TextSystem`) and the
app-global shaped-buffer cache (`TextShaper`) is sound: widget identities are
not window-namespaced, while the cosmic-text buffers are safely shareable.
The main structural problem is instead the absence of one canonical shaping
request and one cohesive ownership boundary. The same inputs are represented
and interpreted by several types and protocols, the low-level implementation
surface is public despite having no production consumer, and editor-only
geometry occupies the shaping/cache root module.

Current flow: layout extracts raw text fields from `ShapeRecord`, creates an
unbounded identity-cached run, and may consume that run in a second bounded
shape step. Cosmic shaping converts those inputs into a reconstructible
`TextCacheKey`; the backend later combines the key with record-local source
text to restore an evicted buffer. `TextEdit` bypasses the identity cache and
borrows the same global shaper directly for caret, hit-test, and selection
geometry.

## High: one shaping request has several independent representations

- [ ] **Shaping identity is manually duplicated across raw parameters, validated parameters, metrics, and a reversible cache key.** `ShapeParams`, `ValidatedShapeParams`, and `TextMetrics` retain overlapping versions of the same font-size, line-height, width, family, weight, and alignment inputs (`src/text/mod.rs:208`, `src/text/mod.rs:261`, `src/text/mod.rs:290`), while `TextCacheKey` encodes those fields again (`src/text/mod.rs:748`). `key_for`, `canonical_params`, `line_fit`, and `unbounded_key` manually maintain both conversion directions and enum-discriminant mappings (`src/text/cosmic.rs:75`, `src/text/cosmic.rs:109`, `src/text/cosmic.rs:140`, `src/text/cosmic.rs:149`). Any new shaping axis or changed normalization rule must remain synchronized across all of these representations; drift can make layout reuse the wrong buffer or make backend reconstruction disagree with the original shape, with the mismatch detected only by the runtime assertion after reconstruction (`src/text/cosmic.rs:533`).

- [ ] **The identity-cache API models one shape as a staged, borrow-bearing protocol with four coordinator entities.** `TextRunIdentity`, `PreparedTextRun`, `PreparedTextRunState`, and `TextReuseEntry` jointly represent one lookup (`src/text/mod.rs:158`, `src/text/mod.rs:165`, `src/text/mod.rs:174`, `src/text/mod.rs:1080`); `prepare_run` returns a value whose hidden optional state distinguishes empty text and holds a mutable map entry alive (`src/text/mod.rs:426`), and `shape_bounded` must consume that value to complete the request (`src/text/mod.rs:490`). The bounded production caller has to copy `unbounded` before conditionally consuming `prepared` (`src/layout/engine.rs:852`, `src/layout/engine.rs:907`). This protocol inflates entity count and exposes cache sequencing to layout code, so changes to a single shaping operation propagate through several lifetime-coupled types and call-site phases.

- [ ] **Width and alignment are fields of `ShapeParams` but are deliberately erased and passed again beside a separate `LineFit`.** `ValidatedShapeParams::unbounded` resets `max_width_px` and `halign` (`src/text/mod.rs:266`), `prepare_run` always applies that reset (`src/text/mod.rs:426`), and `shape_bounded` later accepts width, alignment, and fit as three additional arguments (`src/text/mod.rs:487`). Direct probing, by contrast, interprets the width and alignment already present in `ShapeParams` (`src/text/mod.rs:353`). The same type therefore means different things on the direct and identity-cached paths, and callers must know which fields are honored, discarded, or supplied out of band.

- [ ] **`TextWrap` semantics are decomposed independently by shaping, desired-size, and intrinsic-size code.** Six public policies live in `TextWrap` (`src/text/wrap.rs:11`), layout separately maps them to `LineFit`, decides whether shaping is bounded, adjusts the target width, and special-cases scroll desired size (`src/layout/engine.rs:880`, `src/layout/engine.rs:887`, `src/layout/engine.rs:893`, `src/layout/engine.rs:927`), while intrinsic layout performs another exhaustive interpretation (`src/layout/intrinsic/mod.rs:306`). A policy change can therefore alter painted glyphs without the matching min-content, max-content, or desired-size behavior, producing disagreement between measurement and rendering.

## Medium: boundaries expose or own the wrong responsibilities

- [ ] **The validated text-metrics invariant is discarded between recording and layout, forcing fallible validation back into the trusted hot path.** `Shapes::add` drops every no-op shape before storage (`src/scene/shapes/mod.rs:71`), and text no-op detection already validates font size and line height (`src/shape/mod.rs:304`), yet `ShapeRecord` and `TextShapeInput` store those values as raw `f32`s again (`src/scene/shapes/record.rs:104`, `src/layout/support.rs:27`). `TextSystem::prepare_run` consequently returns `Result` and layout immediately `expect`s success on every text run (`src/text/mod.rs:426`, `src/layout/engine.rs:852`). The same malformed values are silently omitted at authoring, reported as `ShapeParamsError` by the public direct API, and treated as impossible inside layout, while the extra validation/result types remain threaded through core code.

- [ ] **The public API exposes an implementation-level shaping stack that production code does not use.** The crate root exports `CosmicMeasure`, `ShapeParams`, `ShapeParamsError`, and `TextMeasurement` alongside `TextShaper` (`src/lib.rs:122`); `CosmicMeasure` has private state and its only public construction path loads the same bundled fonts as `TextShaper::with_bundled_fonts`, with `TextShaper::with_cosmic` as the only public bridge (`src/text/cosmic.rs:215`, `src/text/cosmic.rs:250`, `src/text/mod.rs:317`). The public `TextShaper::measure` method is bypassed by layout and `TextEdit`, which use crate-private coordinator/probe APIs (`src/text/mod.rs:339`, `src/layout/engine.rs:852`, `src/widgets/text_edit/view.rs:228`); production hosts either construct the bundled shaper or merely accept a `TextShaper` handle (`src/host/winit/mod.rs:199`, `src/host/offscreen.rs:121`). Consumers are thus asked to understand several cache/measurement entities that add no capability used by Aperture itself and bind the supported API to cosmic-text implementation details.

- [ ] **Editor-only caret, hit-test, and selection geometry occupies the shaping/cache root module.** `TextLayoutProbe`, `CursorPos`, `SelectionRects`, cursor conversion, mono hit-testing, and selection-rectangle construction account for the geometry block in `src/text/mod.rs` (`src/text/mod.rs:61`, `src/text/mod.rs:147`, `src/text/mod.rs:541`, `src/text/mod.rs:595`, `src/text/mod.rs:905`, `src/text/mod.rs:976`), but their only production consumer is `TextEdit` (`src/widgets/text_edit/view.rs:13`, `src/widgets/text_edit/input.rs:120`). This makes the generic text cache depend on editor interaction semantics and `unicode-segmentation` (`src/text/mod.rs:35`), while the 1,184-line root mixes public vocabulary, cache ownership, shaping dispatch, editor geometry, placement, and fallback measurement.

- [ ] **Window-local finalization owns maintenance of an app-global buffer cache.** Every `Ui::finalize_frame` calls its own `TextSystem::end_frame` (`src/ui/mod.rs:423`, `src/ui/mod.rs:426`), which immediately runs `TextShaper::end_frame` before sweeping the window-local identity rows (`src/text/mod.rs:399`); that shaper handle is clone-shared by every window and the backend (`src/host/window_driver.rs:219`, `src/host/window_driver.rs:222`, `src/host/shared.rs:34`). Global LRU maintenance is therefore scheduled according to each window's independent frame cadence and interleaved with local widget eviction, so the same global resource can be budgeted multiple times during one multi-window host cycle and its ownership remains split between layout and host/backend lifetimes.

## Low: avoidable hot-path work remains

- [ ] **Identity reuse compares exact raw floats even though the content cache shapes only canonicalized values.** `TextReuseInputs` derives `PartialEq` over `ValidatedShapeParams` and compares it directly on every occupied identity row (`src/text/mod.rs:434`, `src/text/mod.rs:464`, `src/text/mod.rs:1073`), while cosmic-text keys quantize metrics and widths to a 1/64-pixel grid before shaping (`src/text/cosmic.rs:56`, `src/text/cosmic.rs:75`). Two authoring values that map to the same shaped buffer still invalidate the identity row, clear its bounded result, increment dispatch accounting, and borrow the shared shaper, turning a content-cache hit into an avoidable identity-cache miss.

- [ ] **Test-only dispatch accounting is stored and mutated in production shaping state.** `TextShaper::inner` is `pub(crate)` specifically for test observability, and `ShaperInner` unconditionally carries `measure_calls` (`src/text/mod.rs:121`, `src/text/mod.rs:184`); direct measurement, layout probes, identity refreshes, and bounded misses all increment it (`src/text/mod.rs:349`, `src/text/mod.rs:374`, `src/text/mod.rs:448`, `src/text/mod.rs:522`). Production text work pays the shared mutable-state write and carries a wider internal visibility boundary solely for cache-effectiveness tests.
